# Robotaxi 站点验收报告 - 算法逻辑说明

## 1. 概述

本工具用于自动解析 Robotaxi 站点测试数据，并生成包含测试结论、问题分布、站点统计等内容的 Excel 报告。

核心流程：

```
原始测试数据 → 解析 → 统计 → 分析 → 生成 Excel 报告
```

主要文件：

| 文件 | 作用 |
|------|------|
| `src/utils/parser.ts` | 解析 CSV / Excel / JSON / 文本数据，生成标准 `TestRecord` |
| `src/utils/calculator.ts` | 按站点/方向/问题统计，判定站点结论 |
| `src/utils/analyzeReport.ts` | 生成文字分析结论 |
| `src/utils/reportGenerator.ts` | 生成 Excel 报告 |

---

## 2. 数据解析（parser.ts）

### 2.1 输入格式

支持以下文件格式：

- CSV（`.csv`）
- Excel（`.xlsx` / `.xls`）
- JSON（`.json`）
- 文本（自动识别 `\t`、`,`、`|`、`;` 分隔符）

### 2.2 关键字段映射

解析时会将中文/英文列名统一为标准字段：

| 标准字段 | 常见来源列名 |
|---------|------------|
| `stationName` | 站点名称 / stationName |
| `testType` | 测试类型 / 方向 / testType |
| `result` | 结果 / 测试状态 / result |
| `issueCategory` | 问题标签 / 进站标签 / 停泊标签 / 出站标签 |
| `issueDescription` | 问题描述 / 进站问题描述 / 停泊问题描述 / 出站问题描述 |
| `stationConclusion` | 站点状态 / 站点结论 |
| `version` | 版本 / 智驾版本 |
| `vin` | VIN / 车辆VIN |
| `testDate` | 测试日期 / 日期 |
| `sessionId` | 会话ID / sessionId |

### 2.3 测试类型识别

从 `testType` 或 `issueCategory` 文本中识别：

- `进站`
- `停泊`
- `出站`
- `完整站点测试`

### 2.4 结果标准化

把 `OK`、`通过`、`成功`、`Pass` 统一为 `通过`，把 `NG`、`失败`、`不通过`、`Fail` 统一为 `失败`。

### 2.5 版本号提取

从智驾版本字符串中提取 `XX.r` 段，例如：

```
ZRD.V3.0-Robot-TU.Z30.2026.28.r.102310711173205 → 28.r
```

## 3. 测试统计（calculator.ts）

### 3.1 会话分组

按 `sessionId` 分组，一次会话通常包含同一个测试周期内的 `进站`、`停泊`、`出站` 三条记录。

### 3.2 单次测试通过判定

优先读取 `singleResult` 字段：

- `OK` / `通过` / `Pass` / `成功` → 通过
- 其他 → 失败

如果 `singleResult` 为空，则看同一个会话内：

- `进站` 结果为 `通过` 且 `停泊` 结果为 `通过` → 通过
- 否则 → 失败

### 3.3 方向统计

分别统计 `进站`、`停泊`、`出站` 的测试次数、通过次数、失败次数、成功率：

```
成功率 = 通过次数 / 测试次数 × 100%
```

### 3.4 站点结论判定（4次测试法）

按站点名称聚合所有会话，根据以下规则判定：

| 条件 | 结论 |
|------|------|
| 通过会话数 ≥ 3 | 通过 |
| 失败会话数 ≥ 2 | 不通过 |
| 任意会话被标记为 `站点不合理` | 站点不合理 |
| 以上都不满足 | 未完成 |

**注意**：

- 4次测试法完成数 = 通过站点数 + 不通过站点数（不包含站点不合理）
- 站点不合理站点单独统计

### 3.5 站点结论统计

```
整体通过率（4次测试法） = 通过站点数 / (通过站点数 + 不通过站点数) × 100%
```

站点不合理率、未完成率以全部站点为分母。

## 4. 问题统计（calculator.ts）

### 4.1 问题来源

只统计 `result === '失败'` 且 `issueCategory` 不为空的记录。

### 4.2 排除项

`issueCategory === '站点不合理'` 的记录不纳入问题统计，因为站点不合理已在站点结论中单独统计。

### 4.3 问题类别提取

从标签中提取 `/` 后的内容作为类别，例如：

- `进站失败/其他` → `其他`
- `停泊失败/压线停车` → `压线停车`

### 4.4 特殊规则：进站失败/其他

对于 `进站` 测试类型，如果类别为 `其他`，则改用 `issueDescription`（进站问题描述）作为本条问题的记录。

这样可以避免大量进站问题都被归类为模糊的 `其他`。

### 4.5 问题分布

按 `进站`、`停泊`、`出站` 三个方向分别统计问题类别次数和占比：

```
占比 = 该类别次数 / 全局问题总数 × 100%
```

全局问题总数 = 所有方向的有效问题次数之和。

---

## 5. 文字分析（analyzeReport.ts）

根据统计数据自动生成：

- **测试总结**：覆盖站点数、单次通过率、站点整体通过率、整体评价
- **薄弱环节**：成功率最低的测试方向
- **主要问题**：TOP 3 问题类别
- **版本对比**：多版本时对比提升/回退
- **优化建议**：根据薄弱环节和问题给出建议

## 6. Excel 报告生成（reportGenerator.ts）

### 6.1 工作表结构

生成 2 个工作表：

1. **测试报告**
2. **站点统计**

### 6.2 测试报告布局

| 区域 | 内容 |
|------|------|
| 标题栏 | Robotaxi 站点验收报告、智驾版本、验收日期、VIN |
| 测试结论 | 自动生成的文字分析 |
| 测试结果 | 整体通过率、单次通过率、进站/停泊/出站成功率、站点不合理数 |
| 问题分布 | 按方向分类的问题描述、次数、占比、合计 |
| 验收标准 | 站点结论判定规则说明 |

### 6.3 站点统计布局

| 区域 | 内容 |
|------|------|
| 通过站点 | 站点名称列表 |
| 不通过站点 | 站点名称 + 失败原因 |
| 站点不合理 | 站点名称 + 不合理原因 |

### 6.4 样式说明

- 标题：深灰色背景 + 白字
- 副标题：浅灰色背景 + 深灰字
- 区块标题：浅蓝色背景 + 深灰字
- 表头：深灰色背景 + 白字
- 数据行：白色 / 浅灰色交替
- 成功率/次数列：蓝色渐变数据条（条件格式）

## 7. 最近调整记录

1. **智驾版本提取**：只保留 `28.r` / `30.r` 等字段。
2. **问题分布**：动态行数，所有问题类别都展示。
3. **Excel 样式**：按模板统一配色、合并单元格、百分比公式。
4. **问题总数**：从问题分布表头移除，只在底部合计行显示。
5. **问题分布布局**：合并 `问题描述` 列，`次数` 和 `占比` 右移，方向标签加浅蓝色背景。
6. **数据条**：成功率列和次数列添加蓝色数据条。
7. **站点不合理**：在测试结果中单独统计一行。
8. **站点统计页**：新增通过/不通过/不合理站点及原因。
9. **4次测试法完成数**：只包含通过 + 不通过站点，不含站点不合理。
10. **问题排除**：站点不合理不再计入问题分布。

## 8. 数据流示例

原始记录：

```
站点A, 进站, 失败, 进站失败/其他, 没有靠边停车
站点A, 停泊, 通过, , 
站点A, 出站, 通过, , 
```

处理结果：

- 单次测试：进站失败 + 停泊通过 → 失败（因为 singleResult 为空）
- 问题类别：因为 `进站` + `其他` → 用描述 `没有靠边停车` 作为类别
- 站点结论：根据所有会话判定（如 3 通过/2 失败/不合理 等）
